本报告覆盖项目说明文档要求的三条主线：

- AIGC 辅助生产：Prompt、素材、文本生成、配图生成、发布后二次编辑。

- 内容分发与优化：热榜排序、无限滚动、阅读量反馈、模拟平台分发。

## AI 生成效果

当前文本生成要求模型返回结构化 JSON：

- `title`
- `content`
- `suggested_tags`

优势：

- 前端可以稳定填充标题和正文。
- 失败时有本地兜底内容，演示不会因为密钥缺失中断。
- Prompt 模板覆盖信息增量型短图文、种草内容、长文大纲三个常见场景。

不足：

- Prompt 模板仍是静态数组，没有后台管理。
- 没有保存用户自定义 Prompt。
- 生成内容没有多轮调优记录，难以展示 PE 调参过程。

优化方向：

- 建立 `prompts` 表，支持模板分类、收藏、版本号。
- 保存每次生成请求和响应，用于复盘 Prompt 效果。
- 增加“改写”“扩写”“标题优化”“摘要生成”等轻量二次创作动作。

## 安全审核效果

当前审核覆盖涉黄、涉赌、涉毒、政治敏感、暴力恐怖、个人隐私、未成年人风险、虚假夸大营销等维度。

优势：

- 发布路径强制审核，风险内容会被后端驳回。
- 前端可展示风险等级、原因、置信度和替代文本。
- 审核结果写入 `audit_logs`，有基础留痕能力。

不足：

- 还没有人工标注样本集，无法证明 90% 高危识别准确率。
- 图片、视频、音频素材没有多模态安全审核。
- 对“边界内容”的误杀率没有评估数据。

优化方向：

- 构建 300-500 条标注样本，覆盖正常、高危和边界内容。
- 输出 Precision、Recall、F1 和高危 Recall。
- 将审核日志用于抽样复审，形成规则迭代闭环。

## 质量评分效果

质量评分用于分发排序，维度包括结构、深度、流畅度等。

优势：

- 评分结果持久化到文章表。
- 热榜公式使用质量分，避免只按时间或阅读量排序。
- 本地兜底评分保证无 AI 密钥时也能发布和演示。

不足：

- 质量评分解释没有在前端完整展示。
- 评分和最终阅读表现之间没有相关性分析。

优化方向：

- 在创作工作台展示结构、深度、流畅度子分。
- 按文章质量分区间统计阅读量、点击率和发布成功率。
- 引入人工反馈或点赞收藏作为排序信号。

## 分发排序效果

当前热榜公式：

```text
score = quality_score * 0.4 + ln(view_count + 1) * 0.4 - age_hours * 0.2
```

优势：

- 兼顾内容质量、阅读热度和发布时间。
- 使用 Redis zset 支撑快速分页。
- Redis 无数据时回退 MySQL 已发布文章。

不足：

- 没有外部热点数据源。
- 没有用户反馈信号，如点赞、收藏、负反馈。
- 游标使用分数分页，极端同分场景可能不够稳定。

优化方向：

- 增加二级排序键，如文章 ID 或发布时间。
- 引入用户行为：停留时长、点赞、收藏、负反馈。
- 接入真实热点 API 或维护热点关键词池。
