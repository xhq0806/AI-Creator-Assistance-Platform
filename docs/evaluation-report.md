## AI 生成效果

当前 AI 接入方式以火山方舟 REST API 为主：

- 文本生成、审核、质量评分：`/api/v3/responses`
- 配图生成：`/api/v3/images/generations`，模型配置为 Doubao-Seedream-4.0
- 视频生成：预留 `/api/v3/responses` 调用入口，依赖 `ARK_VIDEO_MODEL`

文本生成要求模型返回结构化 JSON：

```json
{
  "title": "标题",
  "content": "正文",
  "suggested_tags": ["标签"]
}
```

优势：

- 前端可以稳定填充标题和正文。
- 失败时有本地兜底内容，演示不会因为外部 API 临时失败中断。
- Seedream 配图已通过专用图片接口接入，图片与提示词、标题和正文的相关性明显优于占位图方案。

不足：

- 文本生成结果的多轮调优记录还没有持久化。
- 视频生成已接 API 入口和创作页按钮，但效果依赖视频模型配置，仍需补充实际视频模型测试。

## 分发排序效果

当前热榜公式：

```text
score = quality_score * 0.4 + ln(view_count + 1) * 0.4 + ln(like_count + favorite_count * 2 + 1) * 0.3 - negative_count * 0.3 - age_hours * 0.2
```

优势：

- 兼顾内容质量、阅读热度、用户反馈和发布时间。
- 使用 Redis zset 支撑快速分页。
- Redis 无数据时回退 MySQL 已发布文章。

不足：

- 没有外部热点数据源。
- 已加入点赞、收藏、负反馈信号，但还没有防刷和用户级去重。
- 游标使用分数分页，极端同分场景可能不够稳定。

优化方向：

- 增加二级排序键，如文章 ID 或发布时间。
- 引入更多用户行为：停留时长、评论、分享。
- 接入真实热点 API 或维护热点关键词池。
