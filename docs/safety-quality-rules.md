# 内容安全审核规则与质量评估体系

## 审核目标

内容安全模块作为发布前的防线，目标是在不打断正常创作的前提下识别高风险内容，并给创作者一个可执行的修正路径。系统当前采用大模型结构化审核，模型不可用时使用本地兜底结论保障流程可运行。

## 审核维度

| 维度 | 风险示例 | 处理策略 |
|---|---|---|
| 涉黄低俗 | 情色交易、露骨性描写、低俗引流 | 高危拦截，中低危建议改写 |
| 涉赌 | 赌博平台、赌资交易、下注技巧 | 拦截 |
| 涉毒 | 毒品购买、制毒、吸毒教学 | 拦截 |
| 政治敏感 | 政治谣言、煽动性表达、敏感事件不实传播 | 拦截或要求中性改写 |
| 暴力恐怖 | 伤害教学、恐怖主义宣传、血腥细节 | 拦截 |
| 个人隐私 | 手机号、身份证、住址、未授权个人信息 | 提醒脱敏或拦截 |
| 未成年人风险 | 诱导未成年人危险行为、侵害未成年人内容 | 拦截 |
| 虚假夸大营销 | 绝对化疗效、虚假承诺、欺诈引流 | 要求改写 |

## 审核返回结构

后端要求 AI 只返回可解析 JSON：

```json
{
  "is_compliant": true,
  "risk_level": "LOW",
  "risk_category": "",
  "reason": "未发现明显风险",
  "accuracy": 0.9,
  "safe_alternative": ""
}
```

字段含义：

- `is_compliant`：是否允许发布。
- `risk_level`：`LOW`、`MEDIUM`、`HIGH`。
- `risk_category`：命中的风险类别。
- `reason`：给用户和平台的解释。
- `accuracy`：模型对判断的置信度。
- `safe_alternative`：不合规时给出的合规替代文本。

## 干预策略

| 条件 | 系统行为 |
|---|---|
| `is_compliant=true` | 允许发布，进入质量评分 |
| `is_compliant=false` | 后端将文章状态置为 `rejected`，发布接口返回 422 |
| 存在 `safe_alternative` | 前端显示“一键应用合规替代内容” |
| AI 无密钥或返回异常 | 启用本地兜底审核，标记原因，保证演示不中断 |

## 质量评估标准

质量评分用于热榜排序，不直接替代安全审核。当前评分区间为 0-100 分。

| 维度 | 权重建议 | 判断标准 |
|---|---:|---|
| 结构完整性 | 25% | 标题、开头、主体、结尾是否清晰 |
| 信息增量 | 30% | 是否提供事实、案例、方法或独立观点 |
| 表达流畅度 | 20% | 语言是否通顺、段落是否自然 |
| 平台适配度 | 15% | 是否适合图文内容消费场景 |
| 互动与传播性 | 10% | 是否有钩子、冲突点或行动建议 |

AI 评分返回结构：

```json
{
  "quality_score": 82,
  "structure": 80,
  "depth": 84,
  "fluency": 86,
  "reason": "结构完整，信息密度较高，表达流畅"
}
```

## 排序使用方式

发布成功后，文章质量分被写入 `articles.quality_score`，并刷新 Redis 热榜。热榜排序公式：

```text
score = quality_score * 0.4 + ln(view_count + 1) * 0.4 + ln(like_count + favorite_count * 2 + 1) * 0.3 - negative_count * 0.3 - age_hours * 0.2
```

设计意图：

- 高质量文章获得基础曝光。
- 阅读量高的文章获得热度加成。
- 点赞和收藏提供正向反馈，收藏权重更高。
- 负反馈会降低热榜权重。
- 老内容随时间自然衰减，给新内容机会。

## 评估方法

为了证明高危问题识别准确率，建议维护一份人工标注样本集：

| 样本类型 | 建议数量 | 目标 |
|---|---:|---|
| 正常创作内容 | 100 | 控制误杀 |
| 黄赌毒样本 | 100 | 高危识别 |
| 隐私泄露样本 | 50 | 脱敏识别 |
| 虚假营销样本 | 50 | 夸大表达识别 |
| 边界表达样本 | 50 | 校验改写建议 |

指标：

- Precision：判为违规的内容中，真正违规的比例。
- Recall：所有违规内容中，被识别出来的比例。
- F1：Precision 与 Recall 的综合指标。
- 高危 Recall 目标：不低于 90%。

## 当前实现边界

- 当前没有内置人工标注测试集，准确率需通过后续样本评估证明。
- 素材安全只做 URL、文件类型和基础风险词校验，尚未对图片、视频、音频内容做多模态审核。
- 质量评分依赖模型输出，兜底评分只适合演示，不适合线上分发决策。
